# Trust Messaging

Use wording that matches the current validation evidence.

## Recommended baseline wording

- "Validated in SCORM Cloud for launch, completion, score, pass/fail, and resume."
- "Broader LMS interoperability testing is in progress."
- "Source YAML remains the system of record. SCORM packages are generated build artifacts."
- "Validation records are tracked in the proof center and updated platform by platform."

## Wording to avoid

Avoid claims like:

- "Works in every LMS"
- "Universal LMS compatibility"
- "Fully validated across all platforms"

Those claims should only be used if the target LMS has a dated validation record in the proof center.

## Outreach-friendly summary

Short version:

> Structured YAML authoring, compiled preview, SCORM 1.2 export, and a visible SCORM Cloud validation baseline.

Longer version:

> LDT Engine is a structured authoring system for branching training modules. It generates SCORM 1.2 packages from validated source and tracks interoperability evidence in a public proof center.

## Beta framing

Recommended beta note:

- structured authoring
- browser preview
- SCORM 1.2 export
- SCORM Cloud validated baseline
- broader LMS testing in progress

Keep the tone factual. The proof center should be the canonical source for validation claims.
